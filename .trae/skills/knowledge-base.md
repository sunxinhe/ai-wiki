# 知识库管理技能

> **知识库根目录：** `/workspace`
> **知识分类：** 基础知识、前沿探索、学科历史、工程应用、随手记

## 触发条件

当用户提出以下需求时自动触发此技能：

**写入知识相关：**
- "记录一下..." 或 "记个笔记"
- "添加知识..." 或 "写入知识库"
- "保存这个知识点"
- "学到了..." 或 "今天学到了"
- "记下来..."

**随机抽取相关：**
- "随机抽一个" 或 "随机介绍一个"
- "随机学习" 或 "随便讲点知识"
- "来个知识点" 或 "抽个知识点"
- "今天学什么" 或 "给我介绍一个知识"
- "随机抽取"

## 知识分类说明

| 分类目录 | 适用场景 | 示例 |
|---------|---------|------|
| `基础知识/` | 基础概念、核心原理、入门知识 | 数据结构基础、算法入门、编程语言基础 |
| `前沿探索/` | 新兴技术、前沿研究、趋势观察 | 大模型最新进展、量子计算、AGI 研究 |
| `学科历史/` | 发展历程、重要人物、里程碑事件 | 计算机发展史、AI 简史、互联网演进 |
| `工程应用/` | 实践经验、技术方案、工具使用 | 系统设计、性能优化、部署运维 |
| `随手记/` | 零散想法、灵感记录、待整理内容 | 思考碎片、灵感、待研究的问题 |

## 核心能力

### 1. 知识写入

将用户提供的知识内容整理成规范的 Markdown 文档，写入对应分类目录。

**执行步骤：**

1. **确定分类**
   - 根据知识内容判断所属分类
   - 如用户明确指定分类，按用户指定
   - 如不确定，询问用户或默认放入 `随手记/`

2. **生成文件名**
   - 使用中文标题，简洁明了
   - 格式：`{标题}.md`
   - 避免特殊字符，使用中文或英文

3. **编写知识文档**
   - 文档结构必须包含：
     ```markdown
     # {标题}
     
     > **分类：** {分类}
     > **标签：** {标签1, 标签2, ...}
     > **创建时间：** {YYYY-MM-DD}
     
     ## 概述
     {简要介绍，1-2 句话概括}
     
     ## 核心内容
     {详细的知识内容，可以分多个小节}
     
     ### {子标题1}
     {内容}
     
     ### {子标题2}
     {内容}
     
     ## 关键要点
     - {要点1}
     - {要点2}
     - {要点3}
     
     ## 延伸思考
     {可选：相关的思考问题或扩展方向}
     ```

4. **写入文件**
   - 将文件保存到对应分类目录下
   - 确认文件创建成功

**输出格式：**
```
✅ 知识已写入知识库

📁 分类：{分类}
📄 文件：{文件名}
🏷️ 标签：{标签列表}

📝 内容摘要：
{简要摘要}
```

---

### 2. 随机抽取知识点

从知识库中随机选择一篇文章，提取其中一个关键知识点进行介绍。

**执行步骤：**

1. **扫描知识库**
   - 遍历所有分类目录
   - 收集所有 `.md` 文件路径
   - 排除空文件和模板文件

2. **随机选择文章**
   - 从所有文章中随机选择一篇
   - 如果知识库为空，提示用户先添加知识

3. **提取知识点**
   - 读取选中的文章内容
   - 从"关键要点"中随机提取一条
   - 或者从正文中选取一个核心概念
   - 结合上下文给出完整介绍

4. **展示知识点**
   - 说明文章来源（分类、标题）
   - 介绍选中的知识点
   - 提供简要的背景说明

**输出格式：**
```
🎲 随机知识点抽取

📚 来源：{分类} / {文章标题}
🏷️ 标签：{标签}

💡 知识点：{知识点标题}

{详细介绍内容，2-3 段说明}

---
想要了解更多？可以说"详细讲讲这篇"或"再抽一个"
```

---

### 3. 批量写入

当用户提供多条知识或一大段内容时，可以批量处理。

**执行步骤：**

1. **内容拆分**
   - 将大段内容按主题拆分成独立知识点
   - 每条知识点单独成篇

2. **分类归档**
   - 为每条知识点确定分类
   - 生成相应的标签

3. **批量创建**
   - 依次创建每篇文档
   - 记录创建结果

**输出格式：**
```
📦 批量写入完成

成功写入：{N} 篇
- [x] {分类1}/{标题1}.md
- [x] {分类2}/{标题2}.md
- [x] ...

失败：{M} 篇（如有）
```

---

### 4. 列出知识

展示知识库中的文章列表，支持按分类查看。

**执行步骤：**

1. **统计分类**
   - 统计每个分类的文章数量
   - 计算总文章数

2. **列出来源**
   - 按分类列出文章标题
   - 显示标签和创建日期（可选）

**输出格式：**
```
📚 知识库概览

总计：{N} 篇文章

📁 基础知识 ({n1} 篇)
  - {标题1}
  - {标题2}

📁 前沿探索 ({n2} 篇)
  - {标题1}
  ...
```

## 交互模式

### 模式 1：快速写入
用户直接说内容，自动判断分类并写入。

**示例：**
```
用户：记一下，Transformer 是一种基于自注意力机制的神经网络架构
AI：[自动分类 -> 基础知识，生成文档，确认写入]
```

### 模式 2：指定分类写入
用户明确指定分类。

**示例：**
```
用户：往工程应用里加一篇关于 Docker 容器化部署的笔记
AI：[按指定分类创建文档]
```

### 模式 3：随机抽取
用户想要随机学习。

**示例：**
```
用户：随机抽一个知识点
AI：[随机选择文章，提取要点，进行介绍]
```

### 模式 4：连续抽取
用户想继续抽取下一个。

**示例：**
```
用户：再来一个
AI：[重新随机抽取，避免与上一篇重复（如果可能）]
```

## 约束与规则

1. **内容质量**
   - 写入的内容必须结构清晰、易于理解
   - 关键要点必须准确、精炼
   - 标签要能准确反映内容主题

2. **分类准确性**
   - 优先按用户指定的分类
   - 内容横跨多个分类时，选择最主要的分类
   - 不确定时可以询问用户

3. **文件命名**
   - 文件名要简洁，能体现内容主题
   - 使用中文标题，长度控制在 10-30 字
   - 避免与已有文件重名（重名时加编号或修改名称）

4. **抽取公平性**
   - 尽量保证随机性
   - 连续抽取时尽量避免重复
   - 如果知识库文章较少，说明情况

5. **安全约束**
   - 只读取和写入知识库目录内的文件
   - 不删除现有文件（除非用户明确要求）
   - 重要操作前确认用户意图

## 示例对话

**示例 1：写入知识**
```
用户：记一下，CAP 定理指的是在分布式系统中，一致性（Consistency）、可用性（Availability）和分区容错性（Partition tolerance）三者不可兼得。

AI：✅ 知识已写入知识库

   📁 分类：基础知识
   📄 文件：CAP定理.md
   🏷️ 标签：分布式系统, 理论基础, CAP

   📝 内容摘要：
   CAP 定理是分布式系统中的重要理论，指出一致性、可用性和分区容错性三者无法同时满足，最多只能同时满足两个。
```

**示例 2：随机抽取**
```
用户：随机抽个知识点

AI：🎲 随机知识点抽取

   📚 来源：基础知识 / CAP定理
   🏷️ 标签：分布式系统, 理论基础, CAP

   💡 知识点：CAP 定理的三选二

   CAP 定理是分布式系统领域的基石理论，由计算机科学家 Eric Brewer 提出。
   它指出在一个分布式系统中，一致性（Consistency）、可用性（Availability）
   和分区容错性（Partition tolerance）这三个基本属性，最多只能同时满足两个，
   不可能三者兼得。

   常见的组合策略有：
   - CP：牺牲可用性，保证强一致性（如 Zookeeper）
   - AP：牺牲强一致性，保证高可用（如 Cassandra）
   - CA：在没有网络分区的情况下同时满足（单机系统）

   ---
   想要了解更多？可以说"详细讲讲这篇"或"再抽一个"
```

用户：往学科历史里加一个，关于图灵测试的由来

AI：✅ 知识已写入知识库

   图灵测试由艾伦·图灵在1950年的论文《计算机器与智能》中提出，
   用于判断机器是否具有智能的经典测试方法。
