# 提示工程

## 定义

**提示工程（Prompt Engineering）** 是设计和优化输入提示词，以获得AI模型更好输出的技术和方法。好的提示可以显著提升AI输出的质量、准确性和实用性。

## 提示的基本结构

```
┌─────────────────────────────────────┐
│  系统提示（System Prompt）           │
│  定义AI的角色和行为规则             │
├─────────────────────────────────────┤
│  上下文（Context）                  │
│  提供背景信息和支持性内容           │
├─────────────────────────────────────┤
│  用户输入（User Input）             │
│  具体的任务或问题                   │
├─────────────────────────────────────┤
│  输出格式（Output Format）          │
│  指定期望的输出格式                 │
└─────────────────────────────────────┘
```

## 核心策略

### 1. 清晰具体的指令

```
❌ 差：写关于狗的东西
✅ 好：写一篇300字的短文，介绍金毛犬的性格特点、饲养注意事项和适合的家庭类型
```

### 2. 使用分隔符

```
将以下文本总结为3个要点：
---
这里是被总结的长文本内容...
---
```

### 3. 指定输出格式

```
请以JSON格式返回，字段包括：name, age, occupation
```

### 4. Few-shot示例

```
将以下句子改写成正式语气：

示例1：
输入：这事太牛了！
输出：这件事令人印象深刻。

示例2：
输入：帮我搞一下这个
输出：请协助我处理这项事务。

现在请改写：
输入：快点告诉我答案
输出：
```

## 进阶技巧

### 思维链（Chain of Thought）

```
问题：小明有5个苹果，丢了2个，又买了3个，现在有几个？
回答：让我一步步思考...
1. 初始：5个
2. 丢了：5 - 2 = 3个
3. 买了：3 + 3 = 6个
所以小明现在有6个苹果。
```

### 角色扮演

```
你是一位拥有10年经验的高级软件架构师，负责代码审查。
请审查以下代码并指出潜在问题...
```

### 渐进式提问

```
1. 先问：什么是机器学习？
2. 再问：它和深度学习有什么区别？
3. 最后问：能否用具体例子说明？
```

## 常见模式

| 模式 | 用途 | 示例 |
|------|------|------|
| Zero-shot | 直接提问无需示例 | "翻译：Hello" |
| One-shot | 给一个示例 | 见Few-shot |
| Few-shot | 给多个示例 | 见Few-shot |
| CoT | 展示推理过程 | 见思维链 |

## 注意事项

1. **长度控制**：提示过长可能丢失重点
2. **指令冲突**：避免相互矛盾的指示
3. **边界明确**：明确限制AI的行为范围
4. **迭代优化**：根据输出效果调整提示

## 常见错误

```
❌ 模糊指令：写得好一点
✅ 具体标准：使用专业术语，不超过200字

❌ 开放式结尾：随便写点
✅ 明确格式：按以下列表格式返回：1. 2. 3.

❌ 假设理解：你应该知道我要什么
✅ 提供全部信息：我的需求是...
```
