# 模型量化技术

> **分类：** 工程应用
> **标签：** 模型优化, 量化, 深度学习, 部署
> **创建时间：** 2026-06-25

## 概述
模型量化是一种重要的模型压缩技术,通过降低模型参数的精度来减少模型大小和推理延迟,使大模型能够在资源受限的设备上高效运行。

## 核心内容

### 什么是量化?
量化是将模型参数从高精度表示(如FP32)转换为低精度表示(如INT8、INT4)的过程。这不仅减小了模型体积,还能利用专用硬件加速器提高推理速度。

### 量化方法分类

**1. 训练后量化(Post-Training Quantization, PTQ)**
- 在模型训练完成后进行量化
- 无需重新训练
- 实现简单,但可能有精度损失

**2. 量化感知训练(Quantization-Aware Training, QAT)**
- 在训练过程中模拟量化效果
- 模型学习适应量化带来的误差
- 精度更高,但需要重新训练

### 主流量化技术

**INT8量化**
```
原始: FP32 (32位)
量化后: INT8 (8位)
压缩比: 4倍
```

**INT4量化**
```
原始: FP32 (32位)
量化后: INT4 (4位)
压缩比: 8倍
```

**混合精度量化**
- 关键层使用FP16
- 非关键层使用INT8
- 平衡精度与性能

### 量化实现步骤

**步骤1: 校准(Calibration)**
收集模型在代表性数据集上的激活值分布,确定量化参数。

**步骤2: 量化映射**
将FP32值映射到INT8范围:
```
Q(r) = round(r / scale + zero_point)
```

**步骤3: 反量化**
推理时将INT8转换回FP32:
```
r = (Q(r) - zero_point) * scale
```

### 实际应用案例

**案例1: 移动端部署**
- 原始模型: 1.2GB(FP32)
- INT8量化: 300MB
- 推理速度: 提升2-3倍

**案例2: 边缘设备**
- 原始模型无法运行
- INT8量化后可在边缘设备流畅运行
- 精度损失<1%

## 关键要点
- 量化是模型压缩的核心技术,可实现4-8倍的模型压缩
- PTQ实现简单但可能有精度损失,QAT精度更高但需重新训练
- 校准过程对量化质量至关重要,需要代表性数据集
- 量化技术使大模型能够在移动端和边缘设备上部署
- 混合精度量化可以在精度和性能之间取得平衡
