# AI知识库更新规则

## 更新原则

1. **内容质量优先**
   - 每篇文章需包含：定义、核心概念、实际应用、典型例子
   - 技术术语需有清晰解释，避免过度抽象
   - 内容需与时俱进，反映AI领域最新发展

2. **知识覆盖均衡**
   - 确保各子领域都有代表性文章
   - 新兴领域（如Agent、多模态）优先更新
   - 基础领域定期维护，避免内容过时

3. **元数据规范**
   - 每篇文章需标注：难度等级、标签、更新日期
   - 难度等级：入门(beginner)、进阶(intermediate)、高级(advanced)
   - 标签：多个相关标签，便于检索和抽取

## 更新频率

| 类型 | 频率 | 说明 |
|------|------|------|
| 新增主题 | 每周1-2篇 | 覆盖新概念、新技术 |
| 内容优化 | 每月1次 | 修正错误、补充案例 |
| 整体审视 | 每季度1次 | 检查知识体系完整性 |

## 更新流程

1. **来源筛选**
   - 优先从权威来源获取：arXiv、官方文档、顶级会议论文
   - 关注AI领域重要博客和教程

2. **内容编写**
   - 使用清晰的Markdown格式
   - 包含代码示例时需实际可运行
   - 添加相关图片时使用相对路径

3. **审核检查**
   - 检查文章完整性和准确性
   - 确认元数据正确标注
   - 更新topics-index.json索引

## 知识体系结构

```
AI知识库/
├── fundamentals/          # 基础概念
├── machine-learning/      # 机器学习
├── deep-learning/         # 深度学习
├── llm/                   # 大语言模型
├── agents/               # AI智能体
├── computer-vision/       # 计算机视觉
├── nlp/                  # 自然语言处理
└── reinforcement-learning/# 强化学习
```

## 标签系统

使用统一标签体系：
- 核心标签：ai-basics, ml, deep-learning, nlp, cv, llm, agent, rl
- 扩展标签：transformers, gnn, rlhf, rag, fine-tuning, etc.
